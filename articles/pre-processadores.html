Pré Processadores CSS

CSS BÁSICO

Os pré processadores CSS são programas com interpretadores de código que possuem sua própria sintaxe de linguagem de estilização, baseada em CSS. Ao final do processo de escrita do código, estes programas processam (ou compilam) o código em um arquivo no formato CSS para que a estilização possa ser entendida pelo navegador. É uma forma de dar mais funcionalidades à linguagem de estilização, mantendo a compatibilidade com o navegador.

As vantagens de se utilizar pré processadores são justamente as funcionalidades adicionadas com o intuito de deixar o código mais reutilizável, legível e de melhor manutenção. Estas funcionalidades são: mixins, funções, utilização de operadores matemáticos, aninhamento de seletores e pseudo-classes, condicionais e laços de repetição, criação mais intuitiva de variáveis e herança. Com estas funcionalidades, o processo de escrita de código é bem mais veloz e produtivo, quando em comparação com o CSS puro.

Atualmente, os principais pré processadores são o SASS, o LESS e o Stylus. O SASS foi o primeiro a ser criado, em 2006. Em 2009, o LESS foi criado com o intuito de se aproximar mais da sintaxe do CSS puro, em comparação ao SASS. Logo após, em 2010, o Stylus foi criado com sua própria sintaxe. Os três possuem algumas diferenças entre si, como na declaração de variáveis e de mixins, que podem ser visualizadas, por exemplo, na documentação do Stylus disponível aqui.

Com a constante busca por evolução das técnicas utilizadas, o SCSS foi criado no ecossistema do SASS como uma nova sintaxe, mais aproximada ao CSS, sendo hoje uma das mais utilizadas. As principais diferenças entre o SASS e o SCSS são:

  • Regras de sintaxe: A sintaxe original do SASS é chamada sintaxe identada, se aproximando bastante de como é escrito código em Python, onde definições de blocos de código e aninhamentos são criados por meio da identação. Quanto mais identado, mais interna é a camada do seletor. Já no SCSS, o uso de chaves ({}) para delimitar blocos de código e ponto e vírgula (;) no final de cada linha são obrigatórios, assim como no CSS puro.
  • Integração em projetos com CSS puro existente: A integração do SASS em projetos existentes é mais trabalhosa, visto que, com a sintaxe diferente, muita reescrita de código será necessária. Com o SCSS, é possível apenas criar novas linhas de código para as novas funcionalidades trazidas pelo pré processador, já que a sintaxe do CSS puro é semelhante à do SCSS.
  • Extensão do arquivo: A extensão de arquivos SASS é .sass e a de arquivos SCSS é .scss.
  • Aninhamento hierarquico: Devido à sintaxe, aninhamentos no SASS podem se tornar menos legíveis e de difícil manutenção, a depender do nível de aninhamento criado no HTML. No SCSS, pela organização na sintaxe, esse ponto é melhor abordado.
  • Compatibilidade com CSS puro: um código CSS não pode ser utilizado como SASS e vice versa, ao passo que um código CSS pode ser utilizado como um código SCSS e um código SCSS pode ser utilizado como CSS, a não ser que contenha funcionalidades não suportadas no CSS puro.

Para utilização dos pré processadores, são requeridas algumas instalações e comandos, que podem ser realizados por meio do terminal de comando ou com executáveis. O passo a passo a seguir é uma das formas para instalar o SASS via npm, mas cada pré processador possui, em sua documentação, informações sobre como instalá-lo.

  1. Instalar o NodeJS: O NodeJS é um ambiente de execução de JavaScript no lado do servidor e é necessário para a utilização do npm. Em seu site oficial, é possível fazer o download do instalador e realizar a instalação. É indicado sempre baixar a versão "LTS", visto que é a mais estável e que possuirá maior suporte. É possível, após a instalação, rodar o comando node -v no terminal para verificar se a instalação ocorreu como esperado. Se sim, o terminal irá retornar a versão instalada do NodeJS.
  2. Instalar o npm: O npm é instalado automaticamente quando se instala o NodeJS. No momento da produção deste artigo, a versão LTS mais recente do NodeJS é a 20.11.1 e já inclui na sua instalação o npm versão 10.2.4. É possível, após a instalação, rodar o comando npm -v no terminal para verificar se a instalação ocorreu como esperado. Se sim, o terminal irá retornar a versão instalada do npm.
  3. Instalar o SASS: Com o npm instalado, basta rodar o comando npm install -g sass no terminal de comando sendo utilizado e aguardar a instalação. É possível, após a instalação, rodar o comando sass --version no terminal para verificar se a instalação ocorreu como esperado. Se sim, o terminal irá retornar a versão instalada do SASS.
  4. Criar um arquivo SCSS: Como a criação de qualquer outro arquivo, é necessário apenas criá-lo dentro da pasta do projeto, porém, com a extensão .scss.
  5. Pré processar o arquivo em SCSS e gerar um arquivo CSS: Com o SASS instalado, é possível rodar o comando sass para pré processar o arquivo .scss da seguinte maneira: sass [input] [output], em que o input será o arquivo .scss e o output será o arquivo .css pré processado. É possível, também, rodar o comando com a bandeira --watch da seguinte forma: sass --watch [input] [output]. Dessa forma, o SASS irá observar quaisquer alterações no arquivo de input e irá pré processar automaticamente para o arquivo de output.

Seguindo os passos descritos anteriormente, é possível instalar e utilizar o SCSS em projetos de desenvolvimento. É importante notar que, visto que o SASS foi instalado globalmente com o comando utilizado, não é necessária uma nova instalação para cada projeto. Portanto, para projetos futuros, só é necessário realizar os dois últimos passos descritos para a utilização do SCSS. Vale também ressaltar que, mesmo com a utilização do SCSS, o documento que será linkado no HTML é o CSS gerado pelo pré processamento.
